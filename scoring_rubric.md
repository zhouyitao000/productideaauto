# Product创意 Scoring Rubric

## Overview
This document defines the scoring criteria for evaluating product创意 ideas generated from Weibo hot search topics. The scoring system uses two dimensions: **有趣度 (Interest Score)** and **有用度 (Usefulness Score)**, weighted 80% and 20% respectively.

## 1. Interesting Score (有趣度: 0-100)

### 1.1 Definition
评估创意产品的娱乐性、情感共鸣、社交传播潜力和吸引力。

### 1.2 Scoring Levels

#### 90-100: 卓越娱乐性 (Exceptional)
- **特征**:
  - 高度娱乐性，能引发强烈情感反应（欢笑、惊喜、感动）
  - 病毒式传播潜力极高，用户会主动分享
  - 与热点话题有深刻的情感共鸣
  - 创意新颖独特，令人印象深刻
- **示例**: 结合热门综艺的互动游戏，利用明星效应的社交功能

#### 80-89: 优秀娱乐性 (Excellent)
- **特征**:
  - 有吸引力，用户愿意花时间互动
  - 良好的分享价值，适合社交媒体传播
  - 与热点话题有明确的情感连接
  - 创意有一定的新颖性
- **示例**: 红包故事生成器，根据用户红包记录创建可视化故事

#### 70-79: 良好娱乐性 (Good)
- **特征**:
  - 中等有趣，有一定吸引力
  - 可能在某些用户群体中受欢迎
  - 与热点话题关联度适中
  - 创意实现相对简单
- **示例**: 节日主题的表情包生成器

#### 60-69: 一般娱乐性 (Average)
- **特征**:
  - 轻微有趣，吸引力有限
  - 主要在特定场景下有用
  - 与热点话题关联较弱
  - 创意较为普通
- **示例**: 简单的信息聚合工具

#### Below 60: 缺乏娱乐性 (Poor)
- **特征**:
  - 缺乏娱乐元素，难以吸引用户
  - 无情感共鸣或社交价值
  - 与热点话题关联性差
  - 创意平庸或无新意

### 1.3 Evaluation Factors
1. **情感共鸣**: 能否引发用户情感反应
2. **社交价值**: 是否值得在社交平台分享
3. **新颖性**: 创意是否新鲜、独特
4. **娱乐性**: 是否提供娱乐体验
5. **话题关联**: 与热点话题的紧密程度

## 2. Usefulness Score (有用度: 0-100)

### 2.1 Definition
评估创意产品的实用性、问题解决能力、市场需求和用户价值。

### 2.2 Scoring Levels

#### 90-100: 极高实用性 (Exceptional)
- **特征**:
  - 解决真实存在的用户痛点
  - 明确的市场需求，目标用户清晰
  - 高实用价值，能显著改善用户体验
  - 有商业化潜力，可能产生收入
- **示例**: 解决新能源汽车价格战信息不对称的比价工具

#### 80-89: 优秀实用性 (Excellent)
- **特征**:
  - 满足真实用户需求
  - 良好的实用价值，能解决问题
  - 目标用户明确，市场机会可见
  - 有明确的用例场景
- **示例**: 红包理财小助手，帮助用户管理春节红包收入

#### 70-79: 良好实用性 (Good)
- **特征**:
  - 有一定用处，解决次要问题
  - 实用价值有限，但仍有价值
  - 目标用户相对狭窄
  - 市场机会较小
- **示例**: 节日活动提醒工具

#### 60-69: 一般实用性 (Average)
- **特征**:
  - 边际效用，价值有限
  - 解决的问题不痛不痒
  - 用户需求不强烈
  - 市场竞争激烈
- **示例**: 简单的信息展示应用

#### Below 60: 缺乏实用性 (Poor)
- **特征**:
  - 几乎没有实用价值
  - 解决的问题不存在或无关紧要
  - 无明确用户需求
  - 市场机会渺茫

### 2.3 Evaluation Factors
1. **问题解决**: 是否解决真实用户问题
2. **市场需求**: 目标用户群体大小和需求强度
3. **实用价值**: 对用户生活的实际改善程度
4. **商业化潜力**: 收入模式和市场可行性
5. **可持续性**: 长期价值和使用频率

## 3. Total Score Calculation

### 3.1 Formula
```
综合评分 = (有趣度 × 0.8) + (有用度 × 0.2)
```

### 3.2 Weighting Rationale
- **有趣度权重80%**: 微博热搜话题具有强烈的娱乐性和社交属性，创意产品应优先考虑娱乐价值和传播潜力
- **有用度权重20%**: 实用性确保产品有长期价值，避免纯娱乐的昙花一现

### 3.3 Quality Classification

#### 优秀 (Excellent): ≥ 80分
- **标准**: 综合评分80分及以上
- **特征**: 高娱乐性或有强实用性，市场潜力大
- **报告显示**: 绿色高亮，优秀徽章
- **建议**: 优先考虑开发，投入资源

#### 良好 (Good): 60-79分
- **标准**: 综合评分60至79分
- **特征**: 有一定价值，但需要优化或针对细分市场
- **报告显示**: 黄色高亮，良好徽章
- **建议**: 进一步优化或作为次要项目

#### 需要改进 (Needs Improvement): < 60分
- **标准**: 综合评分低于60分
- **特征**: 价值有限，建议重新构思
- **报告显示**: 红色高亮，需要改进徽章
- **建议**: 重新设计或放弃

## 4. Scoring Examples

### 4.1 优秀创意示例
**创意名称**: 红包故事生成器
- **有趣度**: 85/100 (情感共鸣强，社交分享价值高)
- **有用度**: 70/100 (帮助用户记录春节互动，有一定实用价值)
- **综合评分**: (85×0.8) + (70×0.2) = 68 + 14 = 82
- **评级**: 优秀

### 4.2 良好创意示例
**创意名称**: 红包理财小助手
- **有趣度**: 65/100 (有一定趣味性，但非强娱乐)
- **有用度**: 90/100 (解决真实理财需求，实用性强)
- **综合评分**: (65×0.8) + (90×0.2) = 52 + 18 = 70
- **评级**: 良好

### 4.3 需要改进创意示例
**创意名称**: 简单的热搜排行榜复制
- **有趣度**: 40/100 (缺乏新意，无娱乐价值)
- **有用度**: 50/100 (已有类似工具，无独特价值)
- **综合评分**: (40×0.8) + (50×0.2) = 32 + 10 = 42
- **评级**: 需要改进

## 5. Scoring Consistency Guidelines

### 5.1 Cross-Topic Consistency
- 相同娱乐水平的创意应获得相似的有趣度分数
- 相同实用价值的创意应获得相似的有用度分数
- 避免因话题热度而偏袒评分

### 5.2 Justification Requirements
每个分数必须附有1-2句解释:
- **有趣度解释**: "该创意具有高娱乐性，因为..."或"社交传播潜力有限，因为..."
- **有用度解释**: "解决真实用户痛点，因为..."或"市场需求不明确，因为..."

### 5.3 Bias Avoidance
- 避免个人兴趣影响评分
- 基于客观标准而非主观偏好
- 考虑目标用户群体的普遍反应

## 6. Application in Skill Execution

### 6.1 During Creative Generation
1. 为每个创意分别评估有趣度和有用度
2. 根据上述标准选择合适分数区间
3. 计算综合评分并确定质量等级
4. 记录评分理由供报告使用

### 6.2 In HTML Report
1. 显示每个创意的有趣度和有用度分数
2. 用颜色区分质量等级（绿/黄/红）
3. 在评分条中可视化分数比例
4. 在创意卡上显示质量徽章

### 6.3 For Recommendations
1. 按综合评分排序所有创意
2. 选择前3名作为推荐优先级
3. 在报告中突出显示优秀创意
4. 提供基于评分的开发建议

## 7. Adjustments and Customization

### 7.1 Weight Adjustment
如果用户偏好不同，可调整权重比例:
- **娱乐导向**: 有趣度90% + 有用度10%
- **实用导向**: 有趣度50% + 有用度50%
- **平衡导向**: 有趣度80% + 有用度20% (默认)

### 7.2 Industry Specifics
某些行业可能需要调整标准:
- **娱乐行业**: 提高有趣度权重
- **工具行业**: 提高有用度权重
- **社交行业**: 平衡两者但侧重有趣度

### 7.3 User Feedback Integration
长期可基于用户反馈优化评分标准:
- 收集实际开发结果与评分相关性
- 调整分数区间定义
- 更新评估因素权重
